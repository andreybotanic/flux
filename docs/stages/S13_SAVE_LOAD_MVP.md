# S13 — Save/load MVP

## Цель этапа

Добавить минимальное сохранение и загрузку мира с manifest и scenario steps.


## Требования к реализации

- Создать crate `flux_save`.
- Реализовать сохранение и загрузку мира. Сохранения должны храниться в папке saves, которая не должна быть под контролем git.
- Сохранять:
  - save manifest;
  - world dimensions;
  - seed;
  - tick;
  - слои WorldGrid - для каждого слоя в сохранении должен быть предусмотрен свой блок (чанк, область).;
  - registry signature/hash placeholder.
- Расширить главное меню:
  - добавить кнопки с действиями SaveGame/LoadGame с заранее определенным id сохранения;
- Добавить actions, которые должны быть доступны как через кнопки, так и в сценариях:
  - `SaveGame(id)`;
  - `LoadGame(id)`.
- Из действия CreateWorld полностью убрать текущую генерацию контента в мире.
- Создать пару тестовых сохранений, содержащих разный контент.
- Создать сценарий, поочередно загружающий разные сохранения. 


## Что не входит в этап

В этот этап не входит возможность выбрать имя сохранения при сохранении/загрузке через UI.

Это значит, что для осздания сохранения с определенным именем или дла загрузки такого сохранения в меню должна быть отдельная кнопка. Одна кнопка - одно сохранение.
