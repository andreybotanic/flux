# S12 — Scenario runner with UI, screenshot and diagnostic logs

## Цель этапа

Расширить сценарии до имитации базовой активности игрока в UI и снятия скриншотов.


## Требования к реализации

- Расширить scenario steps:
  - `OpenUi(panel_id)`;
  - `Click(widget_id)`;
  - `WaitTicks(n)`;
  - `WaitSimulationSeconds(seconds)` - при запущенной симуляции ждать определенное время в секундах, с точностью до одного шага симуляции (если шаг занимает 150ms, а ждать надо 1s, то нужно ждать 6 шагов);
  - `PauseSimulation`
  - `WaitRealtime(delay)` - ожидание во время паузы
  - `ResumeSimulation`
  - `PauseSimulation(delay)` - комбинация `PauseSimulation` + `WaitRealtime(delay)` + `PauseSimulation(delay)`
  - `TakeScreenshot(path)`;
  - `AssertUiExists(widget_id)`;
  - управление камерой:
    - `SetCameraPivot(x, y)` - клетка, на которой камера должна центрироваться;
    - `SetCameraZoom(z)` - задание масштабирования.
- Для ручного наблюдения сценария в команду запуска сценария добавить параметр `--scenario-visual-delay-ms <ms>`: он добавляет задержку для визуального наблюдения сценария, но не должен изменять simulation tick. По сути этот параметр после каждого шага сценария должен добавлять `PauseSimulation(ms)`. Этот параметр предполагает, что сценарий запускается в оконном режиме.
- Добавить валидацию сценариев:
  - `WaitTicks` и `WaitSimulationSeconds` возможен только когда мир загружен (т.е. после выполнения действия загрузки мира и до выполнения действия открытия меню): пока открыто меню, таймер стоит на паузе, поэтому никаких тиков нет;
  - `WaitRealtime` возможет только на паузе;
  - управление камерой возможно только при загруженном мире: управлять можно только камерой мира, но не камерой для UI;
  - валидация должна происходить перед запуском сценария, невалидный сценарий не должен запускаться.
- Сценарии должны кликать UI через stable IDs, а не через координаты.
- Сценарий должен создавать артефакты:
  - diagnostic log;
  - screenshot.
- Любые `Assert*` команды сценария должны принудительно завершать сценарий если условие ассерта не выполнилось.

## Ручная проверка

1. Запустить UI smoke scenario.
2. Проверить diagnostic log.
3. Проверить наличие скриншота.
